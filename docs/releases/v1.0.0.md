# Maverything 1.0.0

The 1.0 release. Since 0.1.3, Maverything's search engine was substantially rewritten for
speed, accuracy, and memory — the live-refresh CPU "storm" is gone, ranking now matches
fzf/Everything quality, and the whole index is leaner. All changes are covered by the
in-repo simulation suite (8460 assertions).

## Highlights

**Speed**
- Live-refresh "storm" eliminated: incremental sort-order maintenance means a file-attribute
  change (the dominant filesystem event) is an O(1) no-op instead of re-sorting the whole
  index. Per-batch sort cost dropped from ~56 ms to ~4 ms at 1M entries.
- Folder-size totals update incrementally (3.4 ms → 1.5 µs), directory adjacency moved to a
  compact CSR layout, name ordering uses a radix sort.
- Type-ahead now narrows filtered/multi-term queries from the previous keystroke instead of
  rescanning, and single-character queries are much faster.
- Faster cold start: the snapshot is ~26% smaller (derivable arrays are reconstructed on load)
  and name search is served the instant the index loads, before background warm-up finishes.

**Accuracy (fzf / Everything-quality ranking)**
- camelCase word boundaries, best-occurrence scoring, path-depth tiebreak, and a bounded
  Smith-Waterman fuzzy refinement so tight matches outrank scattered ones.
- Relevance ties break alphabetically; non-ASCII `startwith:` / `endwith:` now match.

**Safety / correctness**
- Regex search runs off the index lock, so a catastrophic-backtracking pattern can no longer
  freeze the UI.
- Numerous concurrency and reconcile fixes hardened by cross-model review.

**Memory**
- Index footprint reduced ~25–30 MB at 1M files (narrower arrays, bitmap flags) with bounded
  search caches.

**Other**
- Bundle identifier is now `kr.kdr.maverything` (was `com.maverything.app`). See the migration
  note below.

## ⚠️ Existing 0.1.x users: manual reinstall required

1.0.0 changes the app's bundle identifier, so macOS treats it as a distinct app and Sparkle
**cannot** auto-update a 0.1.x install to 1.0.0. To upgrade:

1. Download `Maverything-1.0.0.dmg` from this release.
2. Drag the new `Maverything.app` into `/Applications`, replacing the old one.
3. Re-grant **Full Disk Access** to the new app (the permission is keyed to the bundle id):
   `System Settings > Privacy & Security > Full Disk Access`.
4. Your saved settings reset to defaults (preferences are stored under the new bundle id). The
   index rebuilds automatically on first launch.

Auto-update (Sparkle) resumes normally for 1.0.x → later releases.

## First Run

For complete whole-disk results, grant Full Disk Access:

```text
System Settings > Privacy & Security > Full Disk Access
```

Without it, Maverything still runs but macOS limits what it can read.

## Known Limitations

- Full Disk Access is required for complete whole-disk results.
- Network shares may need scheduled rescans (they don't always deliver reliable file events).
- Homebrew cask is not yet Developer ID signed/notarized for the main tap.
