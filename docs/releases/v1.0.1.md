# Maverything 1.0.1

A small follow-up to 1.0.0 with two quality-of-life fixes.

## Changes

- **"Start at login" is now on by default** on a fresh install. A real-time index is only
  useful while the app is running, so Maverything now opts in to launching at login the first
  time you run it from `/Applications`, keeping the index warm so search is instant right after
  a reboot. macOS shows its standard "added a login item" notice, and you can turn it off
  anytime in **Settings ▸ General ▸ Startup** or **System Settings ▸ General ▸ Login Items** —
  turn it off once and Maverything won't re-enable it.
- **The Settings window now closes reliably with ESC.** Previously ESC was swallowed whenever a
  control inside Settings held keyboard focus.

## Updating

- **From 1.0.0:** just replace `Maverything.app` in `/Applications` with this build (same app
  identity — no permissions to re-grant).
- **From 0.1.x:** this is the same bundle-identifier change as 1.0.0, so a manual reinstall is
  required — see the [1.0.0 notes](v1.0.0.md): replace the app, then re-grant Full Disk Access
  (the permission is keyed to the new bundle id).

## First Run

For complete whole-disk results, grant Full Disk Access:

```text
System Settings > Privacy & Security > Full Disk Access
```
